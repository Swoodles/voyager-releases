# Voyager — Distribution System

Production-grade auto-updating desktop app.
No Node.js. No Electron. No manual steps for users.

---

## Architecture

```
┌─────────────────────────────────────────────────────┐
│  User's Machine                                      │
│                                                      │
│  Voyager.exe (PyInstaller)                           │
│    └── launcher.py   → shows splash, checks updates  │
│    └── updater.py    → downloads in background       │
│                                                      │
│  %LocalAppData%\Voyager\                             │
│    ├── app\          ← extracted HTML app            │
│    ├── profile\      ← WebView2 data (localStorage)  │
│    ├── updates\      ← downloaded installers         │
│    └── voyager.log   ← debug log                     │
└─────────────────────────────────────────────────────┘
         ↑ checks on launch
┌─────────────────────────────────────────────────────┐
│  GitHub Releases (your repo)                         │
│    version.json  ← {version, download_url, notes}   │
│    Voyager-Setup.exe  ← latest installer             │
└─────────────────────────────────────────────────────┘
```

## One-Time Setup

### 1. Install build dependencies

```bat
pip install pywebview pyinstaller
```

Download and install Inno Setup 6:
https://jrsoftware.org/isdl.php

Install GitHub CLI (for upload step):
https://cli.github.com/

### 2. Create a GitHub repo for releases

Create a **public** repo, e.g. `YOUR_USERNAME/voyager-releases`

Authenticate GitHub CLI:
```bat
gh auth login
```

### 3. Configure build.py

Open `build.py` and set:
```python
GITHUB_REPO = "YOUR_USERNAME/voyager-releases"
```

The `VERSION_JSON_URL` is calculated automatically from that.

### 4. Add version.json to your repo

```bat
git init
git remote add origin https://github.com/YOUR_USERNAME/voyager-releases.git
git add version.json
git commit -m "init"
git push -u origin main
```

### 5. Add iscc to your PATH

Add this to your system PATH:
```
C:\Program Files (x86)\Inno Setup 6
```

## Shipping an Update

### Normal update (patch bump: 1.2.0 → 1.2.1)
```bat
python build.py
```
That's it. The script:
1. Bumps version in `version.json`
2. Embeds new `index.html` into `launcher.py`
3. Compiles `Voyager.exe` with PyInstaller
4. Packages `Voyager-Setup-1.2.1.exe` with Inno Setup
5. Creates a GitHub release and uploads the installer
6. Pushes updated `version.json` to your repo

Users get the update silently on their next launch.

### Minor or major bump
```bat
python build.py --minor    # 1.2.0 → 1.3.0
python build.py --major    # 1.2.0 → 2.0.0
python build.py --version 1.5.0
```

### Build only (no upload)
```bat
python build.py --no-upload
```

## Update UX (What Users See)

1. User launches Voyager — branded splash screen appears instantly
2. App loads in ~1 second (no install wizard, no build steps)
3. In the background, updater checks `version.json`
4. If update is available, installer downloads silently (~5-30 sec)
5. A subtle notification bar appears at the top:
   ```
   ✦ Voyager 1.2.1 is ready — bug fixes and improvements  [Later] [Restart & Update]
   ```
6. User clicks **Restart & Update** → app closes, new installer runs silently, app relaunches

If user clicks **Later**, the bar disappears. The update is applied on the next launch instead.

## Inno Setup PATH Fix

If `iscc` isn't recognized, add it to PATH permanently:

```
System Properties → Environment Variables → System Variables → Path → Edit
Add: C:\Program Files (x86)\Inno Setup 6
```

Or call it directly in build.py — the script already checks common locations.

## Troubleshooting

**"WebView2 not found"** — User needs the Edge WebView2 Runtime.
Windows 11 has it built-in. Windows 10 users click OK in the prompt and it auto-downloads.

**"Update check failed"** — Check `%LocalAppData%\Voyager\voyager.log`
Usually a network error or wrong GitHub URL in `build.py`.

**App data not persisting** — localStorage is stored in `%LocalAppData%\Voyager\profile\`.
Uninstalling does NOT delete this folder, so user data survives updates.

**Silent install not working** — The installer uses `/SILENT /SUPPRESSMSGBOXES /NORESTART`.
Check that `voyager.iss` has `PrivilegesRequired=lowest` (no UAC = smoother silent install).

## Version JSON Schema

```json
{
  "version": "1.2.1",
  "download_url": "https://github.com/YOU/voyager-releases/releases/latest/download/Voyager-Setup.exe",
  "release_notes": "Bug fixes and improvements",
  "min_version": "1.0.0"
}
```

Host this at a stable URL. The default is the `main` branch of your GitHub repo,
accessible via `raw.githubusercontent.com`. It's cached by GitHub's CDN so it loads fast.
